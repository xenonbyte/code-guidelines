# Accessibility (a11y)

## Hard Constraints (MUST NOT)

- MUST NOT ship an interactive control (button, toggle, menu item) that is unreachable or unusable via keyboard alone; hover-only or mouse-only handlers on essential actions are not acceptable.
- MUST NOT ship an `<img>` or icon that conveys meaning without an `alt` attribute; use an empty `alt=""` only for purely decorative images.
- MUST NOT leave a form control without an associated, programmatically linked label (`<label for>`, `aria-label`, or `aria-labelledby`).
- MUST NOT rely on color alone to convey state or meaning (error, success, required); pair it with a text or icon cue.
- MUST NOT use text/background color combinations below WCAG AA contrast thresholds (4.5:1 for normal text, 3:1 for large text).
- MUST NOT trap keyboard focus inside a dialog/modal without a working escape path (Escape key or an explicit, reachable close control).
- MUST NOT use a non-semantic element (`<div onclick>`) as an interactive control without the matching ARIA role, `tabindex`, and keyboard event handlers.
- MUST NOT ship an interactive control smaller than 24x24 CSS px (or without equivalent spacing to an inline target), absent a listed exception (WCAG 2.2 SC 2.5.8 Target Size Minimum).
- MUST NOT ship prerecorded video without synchronized captions (WCAG 1.2.2).
- MUST NOT block paste or autofill on password/authentication fields, and MUST NOT gate access behind a cognitive-function-test CAPTCHA without an accessible alternative (WCAG 3.3.8 Accessible Authentication).

## Ecosystem Idioms & Conventions

- Prefer semantic HTML elements (`<button>`, `<nav>`, `<main>`, `<header>`) over ARIA roles bolted onto generic containers.
- Manage focus explicitly on route changes and after modal open/close so keyboard and screen-reader users land in the right place.
- Provide a visible focus indicator; never remove the `:focus` outline without a replacement that meets contrast requirements.
- Announce dynamic content changes (toasts, inline validation errors) through `aria-live` regions.
- Verify with an actual screen reader and keyboard-only navigation in addition to automated linting.
